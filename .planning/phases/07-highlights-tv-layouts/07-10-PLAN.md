# Plan 07-10: Audio + Game Cancel Persistence

## Objective

Fix 2 UAT-confirmed issues:
1. Game-end audio tones not playing (Test 24) — browser autoplay policy blocks fresh AudioContext
2. "Spiel beenden" button doesn't persist to DB (ad-hoc) — `*Abbruch()` functions clear client only

## Context

### Issue 1 — Audio tones blocked

`playKDATone()` and `playBKTone()` in `index.html` each create a brand-new `AudioContext`:
```js
function playKDATone() {
  try {
    var ctx = new (window.AudioContext || window.webkitAudioContext)();
    // play...
  } catch(e) {}
}
```

**Problem:** Browsers require a prior user gesture before an `AudioContext` can start in "running" state. A freshly created context on a `socket.io` event (no gesture involved) starts in `"suspended"` state and produces no output.

**Fix:** Keep a single global `AudioContext`. Warm it up (call `resume()`) inside the first user click/touch anywhere on the page. Reuse that warmed context in both tone functions.

```js
var _audioCtx = null;
function getAudioCtx() {
  if (!_audioCtx) {
    try { _audioCtx = new (window.AudioContext || window.webkitAudioContext)(); } catch(e) {}
  }
  return _audioCtx;
}
// Warm on first user interaction
document.addEventListener('pointerdown', function warmAudio() {
  var ctx = getAudioCtx();
  if (ctx && ctx.state === 'suspended') ctx.resume();
}, { once: true });
```

Then in `playKDATone()` / `playBKTone()`: replace `new AudioContext()` with `getAudioCtx()`. If ctx is null or suspended, call `ctx.resume().then(function(){ /* play */ })` to guarantee state is running before scheduling oscillators.

### Issue 2 — Spiel beenden no DB persistence

Current `*Abbruch()` functions (bkAbbruch, nEndM, vgAbbruch, fjAbbruch, ankerAbbruch):
```js
function bkAbbruch() { S.aktSpiel = null; showTab('spiele'); renderAll(); }
```

These only clear the client-side reference. The game row in the DB remains `status = 'active'`. On F5 the server detects an active game and returns it, making it reappear.

**Fix:** Add a `DELETE /api/games/:id` route that marks the game `status = 'cancelled'` and `finished_at = datetime('now')`. Each `*Abbruch()` function calls this endpoint before clearing client state.

Server side — add to `Claude/server/routes/games.js`:
```js
// DELETE /api/games/:id — cancel an active game
router.delete('/:id', requireAuth, (req, res) => {
  const game = db.prepare('SELECT * FROM games WHERE id = ?').get(req.params.id);
  if (!game) return res.status(404).json({ error: 'not found' });
  if (game.status !== 'active') return res.status(409).json({ error: 'not active' });
  db.prepare("UPDATE games SET status = 'cancelled', finished_at = datetime('now') WHERE id = ?").run(req.params.id);
  activeGames.delete(Number(req.params.id));
  res.json({ ok: true });
});
```

Also add `'cancelled'` to the `getActiveGame` query guard in `server.js` so cancelled games don't re-appear on reconnect (the query already filters `status = 'active'` so no change needed there — just confirming).

Client side — in `index.html`, add a shared helper:
```js
async function cancelAktSpiel() {
  if (S.aktSpiel && S.aktSpiel.id) {
    try { await fetch('/api/games/' + S.aktSpiel.id, { method: 'DELETE' }); } catch(e) {}
  }
  S.aktSpiel = null;
}
```

Then replace each `*Abbruch()` body with `cancelAktSpiel().then(function(){ showTab('spiele'); renderAll(); })`.

Affected functions: `bkAbbruch`, `nEndM`, `vgAbbruch`, `fjAbbruch`, `ankerAbbruch`.

**Note:** The KDA "Zurück zur Übersicht" button (`S.aktSpiel = null; showTab('spiele');`) has the same issue. Fix it too.

2. Rewrite `playKDATone()`:
```js
function playKDATone() {
  var ctx = getAudioCtx();
  if (!ctx) return;
  ctx.resume().then(function() {
    var freqs = [523, 659, 784, 1047];
    freqs.forEach(function(f, i) {
      var osc = ctx.createOscillator();
      var gain = ctx.createGain();
      osc.type = 'triangle';
      osc.frequency.setValueAtTime(f, ctx.currentTime + i * 0.2);
      gain.gain.setValueAtTime(0.18, ctx.currentTime + i * 0.2);
      gain.gain.exponentialRampToValueAtTime(0.001, ctx.currentTime + i * 0.2 + 0.18);
      osc.connect(gain);
      gain.connect(ctx.destination);
      osc.start(ctx.currentTime + i * 0.2);
      osc.stop(ctx.currentTime + i * 0.2 + 0.18);
    });
  }).catch(function(){});
}
```
