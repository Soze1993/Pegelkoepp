# Phase 7 — UI Design Contract: Highlights & TV-Layouts

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla JS + CSS custom properties |
| Icon library | none — emoji glyphs only (🏆 💩) |
| Font | Bebas Neue (headings, display, scores) + DM Sans (body, labels) |

No shadcn gate applies: this is a vanilla JS/HTML project with no React/Next.js/Vite stack.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline emoji-to-name gap, badge padding |
| sm | 8px | Between role label and score in Fuchsjagd/Viergewinnt TV panels |
| md | 16px | Default padding inside TV overlay text block, card internal padding |
| lg | 24px | TV overlay vertical rhythm between name and subtitle |
| xl | 32px | TV overlay container padding (all sides) |
| 2xl | 48px | Major section gap in Fuchsjagd/Viergewinnt TV split panels |
| 3xl | 64px | Not used in this phase |

Exceptions:
- TV overlay text block uses `padding: 32px 48px` (xl horizontal, 2xl vertical) to fill the screen impressively
- TV player rows keep existing `padding: 1.5vw 2vw` (viewport-relative — matches existing `.player-row` rule)
- Bilderkegeln loser row left-border accent: 4px (not an 8-point token — fixed border width)

## Typography

### Tablet (index.html)

| Role | Font | Size | Weight | Line Height |
|------|------|------|--------|-------------|
| Body | DM Sans (--fb) | 15px (inherited) | 400 | 1.5 |
| Label | DM Sans (--fb) | 11px | 600 | 1.2 |
| Player name in row | DM Sans (--fb) | 15px | 400 | 1.5 |
| Symbol emoji span | DM Sans (--fb) | 15px (inherits row) | 400 | inherit |

Symbol span: `<span class="player-symbol" aria-label="Kegler des Abends">🏆</span>` or `<span class="player-symbol" aria-label="Bilderkegeln-Verlierer">💩</span>`. No additional font sizing — inherits row size.

### TV Layout Scale (tv.html / tv.js — in-game screens)

Exactly 4 functional size tiers for all in-game TV layouts (Bilderkegeln, Fuchsjagd, Viergewinnt):

| Tier | Name | Font | Size | Weight | Line Height | Used For |
|------|------|------|------|--------|-------------|----------|
| T1 | Label | DM Sans (--fb) | 13px | 600 | 1 | Role labels (FUCHS, JÄGER, TEAM X, TEAM O), throw-list entries |
| T2 | Body | Bebas Neue (--fh) | 28px | 400 | 1 | TV section headings (existing), Jäger hunter names and contrib values |
| T3 | Heading | Bebas Neue (--fh) | 36px | 400 | — | Primary player names across all new TV layouts (Fuchsjagd Fuchs name, Viergewinnt player names) |
| T4 | Display | Bebas Neue (--fh) | 72px | 400 | — | Score numbers dominant at TV distance (fp running total, bkTotal score, VS separator) |

Mapping decisions for collapsed values:
- 20px (Fuchsjagd throw list) → collapsed into T1 (13px) — supporting text, not primary information
- 48px (Viergewinnt player score) → collapsed into T3 (36px) — player names and key supporting numbers share the heading tier
- 64px (VS separator) → collapsed into T4 (72px) — VS is a display-tier element at TV distance

### TV Overlay Scale (end-of-game full-screen takeover only)

These sizes apply ONLY to the end-of-game full-screen overlay (KDA winner, Bilderkegeln loser). They do not apply to any in-game TV layout. They are not part of the 4-tier TV layout scale.

| Element | Class | Font | Value | Weight | Line Height | Context |
|---------|-------|------|-------|--------|-------------|---------|
| Overlay emoji prefix | `.tv-overlay-emoji` | DM Sans (--fb) | 10vw | 400 | 1 | KDA/BK full-screen takeover only |
| Overlay player name | `.tv-overlay-name` | Bebas Neue (--fh) | 15vw | 400 | 1 | KDA/BK full-screen takeover only — primary visual anchor |
| Overlay subtitle line | `.tv-overlay-subtitle` | DM Sans (--fb) | 4vw | 600 | 1.2 | KDA/BK full-screen takeover only |

## Component Inventory

### New TV: End Overlay (shared structure)

Both KDA and Bilderkegeln overlays share the same DOM structure and differ only in accent color.

Visual focal point: `.tv-overlay-name` (15vw Bebas Neue) — the player name is the primary visual anchor of the overlay screen.

```
#game.active (replaceChildren called)
  └── div.tv-end-overlay
        ├── div.tv-overlay-emoji     (Overlay Scale: 10vw, textContent = "🏆" or "💩")
        ├── div.tv-overlay-name      (Overlay Scale: 15vw Bebas Neue, --ac or --red — PRIMARY FOCAL POINT)
        └── div.tv-overlay-subtitle  (Overlay Scale: 4vw DM Sans 600, --txt)
```

CSS (inline via `style.cssText`, textContent-only — XSS safe):

```
.tv-end-overlay:
  width: 100vw
  height: 100vh
  display: flex
  flex-direction: column
  align-items: center
  justify-content: center
  gap: 24px
  background: var(--bg)   /* #0f0f14 — full dark takeover */
  padding: 32px 48px
  box-sizing: border-box
  text-align: center
```

Appearance: instant (no CSS transition or animation). The overlay replaces the game view entirely via `gameEl.replaceChildren(overlayEl)` — consistent with how `renderKDABracket` replaces content.

Auto-dismiss: `setTimeout(() => renderIdle(lastWinner), 10000)` — 10 seconds, then `→ renderIdle`.

**KDA overlay specifics:**
- `tv-overlay-emoji` textContent: `"🏆"`
- `tv-overlay-name` textContent: player name (textContent — XSS safe, never innerHTML)
- `tv-overlay-name` color: `var(--ac)` (#e8b84b amber)
- `tv-overlay-subtitle` textContent: `"— Kegler des Abends!"`
- `tv-overlay-subtitle` color: `var(--txt)`

**Bilderkegeln overlay specifics:**
- `tv-overlay-emoji` textContent: `"💩"`
- `tv-overlay-name` color: `var(--red)` (#e05252)
- `tv-overlay-subtitle` textContent: `"— Bilderkegeln-Verlierer!"`

### New Tablet: Player Symbol Span

Placement: after the player name, separated by one `&nbsp;` (4px visual gap). Never before the name — preserves left-alignment of the name column.

Structure:
```html
<span class="player-name-text">Klaus</span><span class="player-symbol" aria-label="Kegler des Abends"> 🏆</span>
```

The space before the emoji is part of the textContent of the symbol span (leading space = 4px gap). No separate CSS class needed beyond `player-symbol` for targeting.

Styling: inherits parent row font (DM Sans 15px). No border, no background — raw emoji only. This distinguishes it from badge chips (`.badge .bamb/.bred`) which are used for status labels in the game library.

Persistence loading: symbol is absent until the startup DB query resolves. No loading spinner or placeholder shown in the row — the row renders normally, symbol appended once data arrives. If no prior KDA or Bilderkegeln game exists, no symbol is shown.

Does NOT replace existing winner highlight styling in the completed game view (`renderBKSpiel`/`showWinnerBanner`) — those operate on different DOM contexts.

Bilderkegeln loser symbol updates live during an active game: re-renders on every `throw:applied` state update (same cycle as the rest of `renderSpielenTab`). The symbol tracks whoever currently has the lowest bkTotal at each render.

### New TV: Bilderkegeln Layout (`renderBilderkegelTV`)

Extends the generic player list. Same `.player-row` structure as the existing generic renderer. The player with the lowest `bkTotal` score gets a visual loser highlight.

Loser row highlight (inline style applied to the `li.player-row` element):
- `border-left: 4px solid var(--red)` — visible indicator at TV viewing distance
- `background: rgba(224, 82, 82, 0.07)` — subtle tint (matches `#e0525212` pattern from index.html BK renderer)
- `padding-left: calc(2vw - 4px)` — compensate for border so name alignment stays consistent

Updates live on every state change (re-renders via existing `throw:applied` and `undo:applied` event handlers).

Score column: uses existing `.player-score` (Bebas Neue T4/72px --ac) — no change to score display.

### New TV: Fuchsjagd Layout (`renderFuchsjagdTV`)

Split direction: left/right (flex-row). Rationale: 16:9 TV aspect ratio gives each side ample vertical space for name stacking; top/bottom would waste horizontal real estate.

```
#game.active (replaceChildren)
  └── div.fj-tv-layout
        ├── div.fj-fuchs-panel   (flex: 1, left side)
        │     ├── div.fj-role-label  "FUCHS" (T1: 13px DM Sans 600 --mut uppercase)
        │     ├── div.fj-player-name (T3: 36px Bebas Neue --txt)
        │     ├── div.fj-throw-list  (each throw: T1 13px DM Sans --mut, comma-separated)
        │     └── div.fj-score       (T4: 72px Bebas Neue --ac = fp running total)
        ├── div.fj-vs-divider        (1px solid var(--brd), vertical, height: 80vh, align-self: center)
        └── div.fj-jaeger-panel  (flex: 1, right side)
              ├── div.fj-role-label  "JÄGER" (T1: 13px DM Sans 600 --mut uppercase)
              └── (for each jaeger player):
                    div.fj-jaeger-row
                      ├── span.fj-jaeger-name  (T2: 28px Bebas Neue --txt)
                      └── span.fj-jaeger-contrib  (T2: 28px Bebas Neue --mut = sum of that jaeger's throws)
```

Fuchs panel background: `var(--card)` with `border-radius: 12px`. Heading color: `var(--ac)` amber (Fuchs is the "prize" role).
Jäger panel background: `var(--card)`. Heading color: `var(--mut)`.

The `fp` field from Fuchsjagd state is the running score (starts at Fuchs total, decremented by Jäger throws). Display this as "Noch: {fp}" in T4 (Bebas Neue 72px) --ac on the Fuchs side — this is the key game number visible from across the room.

Throw list on Fuchs side: shows `state.fuchs.w` as comma-separated values in T1 (13px DM Sans --mut, e.g., "7, 5"). Not strictly required for TV readability but confirms the starting throw.

### New TV: Viergewinnt Layout (`renderViergewinntTV`)

Two team panels side by side. No grid is shown on TV — the 9x9 grid is a tablet interaction element. TV shows team composition and individual scores.

```
#game.active (replaceChildren)
  └── div.vg-tv-layout  (display:flex; flex-direction:row; gap:32px; height:100vh; padding:2vw)
        ├── div.vg-team-panel.vg-team-x   (flex:1, --tX accent #e05252)
        │     ├── div.vg-team-label   "TEAM X" (T2: Bebas Neue 28px --tX)
        │     ├── div.vg-team-border  (border-left: 4px solid var(--tX))
        │     └── (for each tX member):
        │             div.vg-player-row
        │               ├── span.vg-player-emoji+name  (T3: Bebas Neue 36px --txt)
        │               └── span.vg-player-score       (T3: Bebas Neue 36px --tX)
        ├── div.vg-vs-center  (align-self:center; T4: Bebas Neue 72px --mut; textContent:"VS")
        └── div.vg-team-panel.vg-team-o   (flex:1, --tO accent #5b8dee)
              ├── div.vg-team-label   "TEAM O" (T2: Bebas Neue 28px --tO)
              ├── div.vg-team-border  (border-left: 4px solid var(--tO))
              └── (for each tO member):
                      div.vg-player-row
                        ├── span.vg-player-emoji+name  (T3: Bebas Neue 36px --txt)
                        └── span.vg-player-score       (T3: Bebas Neue 36px --tO)
```

Panel background: `var(--card)` (#22222e) with `border-radius: 12px; padding: 24px`.

Score shown per player: sum of all throws by that player (derived from state — executor must determine score accumulation from `viergewinnt.js` state shape; `tX`/`tO` arrays hold player objects but may not carry running totals — if no score field exists, show throw count as proxy or `—` until game ends).

If game is done (`state.winner`): winning team panel gets `border: 2px solid var(--tX or --tO)` glow effect matching the team color. Losing team opacity drops to 0.6.

Draw state (`state.winner === 'draw'`): both panels at full opacity, no border highlight.

## Integration Points (for Executor)

### TV overlay trigger

`socket.on('game:finished', ...)` in `tv.js` (currently line 23):

Replace:
```js
renderGame(state);
setTimeout(function() { renderIdle(lastWinner || null); }, 3000);
```

With:
```js
var typeKey = data.typeKey;   // enriched payload field (D-04)
renderEndOverlay(typeKey, state, lastWinner);
```

`renderEndOverlay(typeKey, state, lastWinner)` builds and mounts the overlay, then calls `setTimeout(() => renderIdle(lastWinner || null), 10000)`.

### TV game-type dispatch

`renderGame(state)` in `tv.js` (currently line 37) — extend the dispatcher:

```js
function renderGame(state) {
  if (state && state.bracket)       { renderKDABracket(state); return; }    // existing KDA
  if (state && state.typeKey === 'bilderkegel') { renderBilderkegelTV(state); return; }
  if (state && state.typeKey === 'fuchsjagd')  { renderFuchsjagdTV(state); return; }
  if (state && state.typeKey === 'viergewinnt') { renderViergewinntTV(state); return; }
  // ... existing generic player list
}
```

Note: `state.typeKey` must be present on the state object for this dispatch to work, OR the executor derives game type from state shape (state.fuchs → fuchsjagd, state.tX → viergewinnt, state.gewinner without bracket → bilderkegel). Either approach is acceptable; explicit `typeKey` on state is preferred for clarity per D-04.

### Backend enrichment

`server/routes/games.js` `game:finished` emission: add `typeKey: game.type_key` to the payload so both the TV and tablet receive it without state-shape inference.

### Tablet symbol injection

In `renderSpielenTab()` → each player-type renderer (at minimum `renderNSpiel`, `renderBKSpiel`, `renderFJSpiel`, `renderVGSpiel`, `renderKDASpiel`): after rendering each player's name, append the symbol span if that player matches the persisted champion/loser IDs. The persisted IDs are loaded from a new API endpoint (`GET /api/highlights/current`) that the page fetches on load.

XSS note: `showWinnerBanner` in index.html currently uses `innerHTML` with `winnerName` derived from DB (line 512). This must be refactored to use safe DOM construction (`textContent` for name). The new overlay and symbol code must exclusively use `textContent` — never `innerHTML` with DB-sourced strings.
