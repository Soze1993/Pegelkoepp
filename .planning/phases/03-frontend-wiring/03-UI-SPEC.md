## New UI Elements тАФ Interaction Contracts

### 1. PIN Login Overlay (D-05)

**Trigger:** `DOMContentLoaded` тЖТ `GET /api/auth/status` returns `authenticated: false`.

**Structure:**
- Full-screen fixed overlay covering entire viewport (`position: fixed; inset: 0`)
- Background: `var(--bg)` at full opacity (not semi-transparent тАФ blocks app entirely)
- Z-index: 999 (above everything including modals at z-index: 100)
- Centred card: `var(--card)` background, `1px solid var(--brd)` border, `border-radius: 16px`
- Card dimensions: 280px width, `padding: 24px`, vertically and horizontally centred via flexbox

**Card contents (top to bottom):**
1. Title: "Anmelden" тАФ 28px Bebas Neue, `color: var(--ac)`, `margin-bottom: 16px`
2. PIN input: `type="password"`, placeholder "PIN eingeben", `text-align: center`, `font-size: 20px`, `letter-spacing: 0.2em`, `margin-bottom: 16px`, existing `input` CSS class styles apply
3. Submit button: label "Anmelden", full-width (`width: 100%`), uses existing `.btn .bp` classes
4. Error message (hidden by default): "Falscher PIN" тАФ 12px DM Sans, `color: var(--red)`, `margin-top: 8px`, shown only after failed login attempt

**Interaction states:**
- Initial: overlay visible, PIN input focused automatically via `focus()` on mount
- Submitting: button does not need a loading spinner (shared-PIN app, instant server response)
- Error: input cleared, input re-focused, error message div set to `display: block`
- Success: overlay `remove()`d from DOM, `init()` called, app becomes visible

**Accessibility:** PIN input has `type="password"` (screen-reader and mobile keyboard appropriate).

**Copy strings:**
- Title: "Anmelden"
- Input placeholder: "PIN eingeben"
- Submit button: "Anmelden"
- Error: "Falscher PIN"

### 2. Connection Dot in Topbar (D-07)

**Position:** Top-right of the topbar, placed alongside the existing `<span id="tsub">Bereit</span>` element. Both items wrapped in a `display: flex; align-items: center; gap: 8px` container.

**HTML element:** `<div class="conn-dot" id="connDot"></div>` тАФ identical class name as TV display.

**CSS (copy verbatim from `tv.html` lines 112тАУ122, adapted for inline position):**
```css
.conn-dot {
  width: 10px;
  height: 10px;
  border-radius: 50%;
  background: #888;          /* initial/unknown state */
  flex-shrink: 0;
}
.conn-dot.green { background: var(--grn); }   /* #4caf7d */
.conn-dot.red   { background: var(--red); }   /* #e05252 */
```

Note: TV's `.conn-dot` is `position: fixed; top: 1rem; right: 1rem`. The topbar version is inline (no fixed positioning). Adjust CSS accordingly тАФ do NOT copy the `position: fixed` rule.

**JS pattern (verbatim from `tv.js` lines 12тАУ13):**
```javascript
socket.on('connect',    function() { document.getElementById('connDot').className = 'conn-dot green'; });
socket.on('disconnect', function() { document.getElementById('connDot').className = 'conn-dot red'; });
```

**States:**
- Unknown (before socket connects): `background: #888` (no class modifier)
- Connected: `.conn-dot.green` тАФ `var(--grn)` = `#4caf7d`
- Disconnected: `.conn-dot.red` тАФ `var(--red)` = `#e05252`

**No tooltip or label** тАФ the dot is a status indicator only; the existing `#tsub` "Bereit" text covers the status function for sighted users.

### 3. Winner Banner (D-12)

**Trigger:** `game:finished` socket event OR `POST /api/games/:id/throws` response with `finished: true`. The banner fires only once per game finish (use `S.aktSpiel._finishing` flag to prevent double-fire тАФ see RESEARCH.md Pitfall 8).

**Structure:**
- Full-screen semi-transparent overlay: `position: fixed; top: 0; left: 0; right: 0; bottom: 0`
- Backdrop: `background: rgba(0, 0, 0, 0.75)`, `z-index: 500`
- Centred card: `var(--card)` background, `2px solid var(--ac)` border, `border-radius: 16px`, `padding: 32px`, `text-align: center`

**Card contents (top to bottom):**
1. Trophy emoji: "ЁЯПЖ" тАФ 48px, `margin-bottom: 8px`
2. Winner text: `[WinnerName] hat gewonnen!` тАФ 32px Bebas Neue, `color: var(--ac)`

**Winner name derivation:** `getWinnerName(state, S.aktSpiel.type_key)` тАФ must handle all 9 game types. For game types where `state.winner` exists, use it. For KDA (`state.gewinner`), Fuchsjagd (`state.winner`), and Viergewinnt (`state.done` + which team's score reached 4), derive from type-specific state fields.

**Timing:**
- Banner appears immediately on trigger
- After 2500ms: banner is `remove()`d, `S.aktSpiel` is set to `null`, tab switches to "Spiele", `renderAll()` is called

**Copy string:** `[WinnerName] hat gewonnen!`

### 6. Spiele Tab тАФ Game History List (D-04)

**Trigger:** Tab becomes active OR `renderSpiele()` is called after game finish navigation.

**Data source:** `GET /api/games?status=finished` тАФ returns games sorted most-recent-first.

**Empty state:** No finished games yet.
- Copy: Use existing `.empty` class
- Icon: "ЁЯО│"
- Heading: "Noch keine Spiele"
- Body: "Spiele erscheinen hier, sobald ein Spiel abgeschlossen wurde."

**Non-empty state:** List of game history cards.

**Card structure:** Reuse existing `.stc` card style (`background: var(--card); border: 1px solid var(--brd); border-radius: 12px; padding: 16px; margin-bottom: 12px`).

Each card displays (modelled on existing `.uc` player card layout):

**Date format:** German locale тАФ `new Date(started_at).toLocaleDateString('de-DE')` тЖТ "20.5.2026"

**Winner name:** From `GET /api/games/:id` or from the winner field if the list endpoint returns it. If winner is `null` (should not occur for finished games but handle gracefully), show "Unbekannt".

## Implementation Notes for Executor

1. **Do not modify CSS or HTML structure.** Only JS data operations change. The UI contract here describes new elements that are INJECTED via JavaScript, not added to the static HTML (except the connection dot, which is added to the existing topbar HTML).

2. **Connection dot HTML is the only static HTML addition.** Modify the topbar `<span id="tsub">` wrapper to add the dot alongside it.

3. **PIN overlay is JS-injected** (`document.createElement`). It is not pre-built in the HTML. Reason: it only exists when `authenticated: false` тАФ a pre-built hidden div would render its DOM in DevTools for all users.

4. **Winner banner is JS-injected** on game completion. Same reason as overlay.

5. **All new elements inherit existing CSS custom properties.** No new CSS rules are needed for spacing or color тАФ use inline styles referencing `var(--token)` or existing class names.

6. **The `.conn-dot` CSS class must be added to `index.html`'s `<style>` block** (copy from `tv.html`), with the `position: fixed` rule removed (the topbar version is inline, not fixed).

7. **`#m-st` modal** (custom game type creator): set `display: none` on the trigger button. The modal HTML can remain in the DOM untouched.
