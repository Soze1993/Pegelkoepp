## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla JS + CSS custom properties (established in public/index.html) |
| Preset | not applicable |
| Component library | none — project-internal component classes only |
| Icon library | emoji glyphs only — no external icon library |

## Component Inventory

All components are composed from existing project classes. No new components are defined.

### 1. "Letzter Abend" Recap Card (`renderSpiele()`)

**Placement:** Inserted into `#r-spiele` immediately after the active-abend banner (or after the "▶ Abend starten" button when no abend is active), before the finished-games list.

**Structure (compose from existing classes):**
```
div.card
  div.stitle           "Letzter Abend"
  div.wi               Date row — "📅 DD.MM.YYYY" left + abend name right (--mut)
  div.wi               KDA row — "🏆 Kegler des Abends" left + "{emoji} {name}" right (--txt)
  div.wi               BK row — "💩 BK Verlierer" left + "{emoji} {name}" right (--txt)
  details (toggle)     "▼ Spiele des Abends" — expands to game list
    div.stbl wrapper
      table.stbl
        thead → Spieltyp / Gewinner / Spieler
        tbody → one row per game
```

If `kda_champion` is `null`, KDA row shows "—" in muted color.
If `bk_loser` is `null`, BK row shows "—" in muted color.
If `games[]` is empty, table body shows: `<tr><td colspan="3" class="empty" style="padding:16px;text-align:center">Keine Spiele aufgezeichnet</td></tr>`.

**Data source:** `GET /api/abende/last-summary` — fetched once on `renderSpiele()` call. If endpoint returns `null` (no closed abend), the entire recap card is not rendered.

**Loading state:** Recap card area shows `<div class="empty"><p>Laden…</p></div>` while fetching.

### 2. Year Leaderboard Section (`renderStats()` extension — top of stats page)

**Placement:** Prepended to `#r-stats` before the existing per-player cards. Wrapped in `div.card`.

**Structure:**
```
div.card
  div  (flex row, space-between, align-items:center, margin-bottom:md)
    div.stitle            "Jahres-Leaderboard"     (margin:0)
    select                Year picker              (width:auto, min-width:80px)
  table.stbl
    thead → # / Spieler / Siege
    tbody → one row per player (sorted by wins desc)
      td.rnk[.gold for rank 1]   rank number
      td                         "{emoji} {name}"
      td.snum (right-aligned)    wins count
```

**Year picker:** Native `<select>` populated from `available_years` in API response. Default value: current year. On `change`, re-fetch `/api/stats/year?year={value}` and re-render table only (not the whole stats page).

**Empty state (year has zero wins recorded):** Replace tbody with:
```
<tr><td colspan="3" style="text-align:center;padding:20px;color:var(--mut);font-size:13px">
Keine Spiele in {year} abgeschlossen.
</td></tr>
```

**Loading state during year change:** Show inline spinner text "Laden…" in tbody, same colspan.

### 3. Win Streak Display (per-player card in `renderStats()`)

**Placement:** In each existing per-player `.uc` card, add two streak chips to the existing chips row (after "Pudel%" chip).

**New chips (compose `.chip` class):**
```
div.chip
  div.v  (color: var(--grn))   "{current}"
  div.l                        "Streak"

div.chip
  div.v                        "{longest}"
  div.l                        "Längster"
```

Streak chips use `.v` with `var(--grn)` for current streak only if `current > 0`; otherwise `var(--mut)`. Longest streak always uses default `.chip .v` color (`var(--ac)`).

**Data source:** `GET /api/stats/streaks` — fetched once when stats tab activates, stored in `S.streaks` (map of `player_id → {current, longest}`). Merged into per-player card render. If `S.streaks` is unavailable, chips show "—".

### 4. Head-to-Head Section (`renderStats()` extension — below year leaderboard)

**Placement:** Between year leaderboard card and the existing per-player cards. Wrapped in `div.card`.

**Structure:**
```
div.card
  div.stitle    "Direkt-Vergleich"
  div.fg
    label       "Spieler A"
    select#h2h-a    (options: all non-archived players, sorted by name)
  div.fg
    label       "Spieler B"
    select#h2h-b    (options: same list)
  button.btn.bp (width:100%)    "Vergleichen"
  div#h2h-result  (initially hidden)
```

**Result area (shown after fetch, inside `#h2h-result`):**
```
div  (flex row, gap:md, margin-top:md)
  div  (flex:1, text-align:center)
    div.snum     "{winsA}"
    div  (font-size:12px, color:var(--mut))   "{nameA}"
  div  (flex:0, padding:0 sm, color:var(--mut), align-self:center)   "vs"
  div  (flex:1, text-align:center)
    div.snum     "{winsB}"
    div  (font-size:12px, color:var(--mut))   "{nameB}"
div  (text-align:center, font-size:12px, color:var(--mut), margin-top:sm)
  "{draws} Unentschieden · {total} Spiele zusammen"
```

**Validation:** If player A === player B, show inline error inside `#h2h-result`:
```
<div style="color:var(--red);font-size:13px;text-align:center;padding:8px">
Bitte zwei verschiedene Spieler wählen.
</div>
```

**Loading state:** "Laden…" text inside `#h2h-result` while fetching.

**Empty state (no shared games):**
```
<div style="color:var(--mut);font-size:13px;text-align:center;padding:12px">
{nameA} und {nameB} haben noch keine gemeinsamen Spiele.
</div>
```

### 5. Award Count Chips in Player Profile Modal (`showSpDetail()`)

**Placement:** Append two new chips to the existing chip grid at the bottom of `#m-sp-detail-c`. The grid currently uses `grid-template-columns:repeat(3,1fr)`. Extend to a new row: a second chip row with `display:grid;grid-template-columns:1fr 1fr;gap:8px;margin-top:8px`.

**New chips:**
```
div.chip
  div.v  (color: var(--ac))    "{kdaCount}"
  div.l                        "KdA-Titel"

div.chip
  div.v  (color: var(--red))   "{bkCount}"
  div.l                        "BK Verlierer"
```

`kdaCount` = number of times player has been "Kegler des Abends" (from `S.kdaCounts[id] || 0`).
`bkCount` = number of times player has been BK loser (from `S.bkCounts[id] || 0`).

Both default to `0` if counts are not yet loaded (no loading state needed — counts are pre-loaded with stats tab).

**Data source:** `GET /api/stats/kda-counts` and `GET /api/stats/bk-counts` — fetched once when stats tab activates, stored in `S.kdaCounts` and `S.bkCounts` (maps of `player_id → count`). These are also used when `showSpDetail()` is called from the Spieler tab before the stats tab has been opened; in that case the counts show `0` (graceful degradation, not an error).

## Interaction Contracts

### Stats Tab Initialization

On `showTab('stats')` (or on `renderAll()`):
1. Fetch `/api/stats` (existing — wins/losses/personal-bests).
2. Fetch `/api/stats/year?year={currentYear}` (new — year leaderboard).
3. Fetch `/api/stats/streaks` (new — win streaks) → store in `S.streaks`.
4. Fetch `/api/stats/kda-counts` (new) → store in `S.kdaCounts`.
5. Fetch `/api/stats/bk-counts` (new) → store in `S.bkCounts`.

Fetches 1–5 are parallel (`Promise.all`). Render after all resolve. On partial failure: render available data; show inline "Laden fehlgeschlagen." only for the section whose fetch failed.

### Year Picker Interaction

`select#year-picker onChange`:
- Set loading text in leaderboard tbody.
- Fetch `/api/stats/year?year={selectedYear}`.
- Replace leaderboard tbody rows only (do not re-render entire stats tab).
- Debounce: not needed (select fires once on user action, not on keystroke).

### H2H Fetch

"Vergleichen" button `onclick`:
- Validate A !== B (inline error if same).
- Show "Laden…" in `#h2h-result`.
- Fetch `/api/stats/h2h?a={playerA}&b={playerB}`.
- Render result or empty state.

### Player Profile Modal Award Counts

`showSpDetail(id)` reads `S.kdaCounts[id]` and `S.bkCounts[id]` synchronously. No fetch on modal open. Counts are 0 if maps not yet populated.

### Recap Card Fetch

Fetched inside `renderSpiele()`, parallel with the existing abend/games fetches. If `GET /api/abende/last-summary` returns `null`, no card is rendered. No retry logic — next `renderSpiele()` call retries naturally.
