  <name>Task 3: Add uploadPhoto() global function to index.html</name>
  <files>public/index.html</files>
  <action>
Find the `// Spieler` comment block in public/index.html (around line 831). Add a new global function `uploadPhoto` BEFORE the `renderSpielerListe` function definition. The function must be at global scope (not inside another function) because it is called via `onchange="uploadPhoto(id, this)"` in the innerHTML string.

Add this function in the script section:

```js
async function uploadPhoto(playerId, inputEl) {
  var file = inputEl.files[0];
  if (!file) return;
  try {
    var res = await fetch('/api/players/' + playerId + '/photo', {
      method: 'POST',
      headers: { 'Content-Type': file.type },
      body: file
    });
    if (res.ok) {
      notify('Foto gespeichert');
      showSpDetail(playerId);  // re-render modal with cache-busted URL
    } else if (res.status === 413) {
      notify('Datei zu groß (max. 5 MB)', 'var(--red)');
    } else {
      var data = {};
      try { data = await res.json(); } catch(_) {}
      notify(data.error || 'Fehler beim Hochladen', 'var(--red)');
    }
  } catch (e) {
    notify('Netzwerkfehler beim Hochladen', 'var(--red)');
  }
}
```

Why `body: file` (not FormData): express.raw() on the server parses the body only when Content-Type matches `image/jpeg` or `image/png`. FormData changes Content-Type to `multipart/form-data` which express.raw() ignores, leaving req.body empty (RESEARCH.md Pattern 3 and Pitfall 3 — verified).

Why `showSpDetail(playerId)` after success: re-renders the modal with a fresh `Date.now()` cache-buster in the img src, immediately showing the uploaded photo (RESEARCH.md Pitfall 5 — browser caches 404).

The `notify()` function is already defined globally in index.html (used throughout the file). Use it as-is.
