<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated tablet client → POST /:id/throws | Session-protected; meta is user-supplied and must round-trip safely through JSON.stringify |
| Authenticated tablet client → POST /:id/undo | Session-protected; the only deletion endpoint in the game lifecycle |
| Server emit → all sockets in room game:${gameId} | Read-only push; payload is server-computed state, never echoed user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | POST /api/games/:id/undo (unauthorized undo) | mitigate | `requireSession` middleware is the first argument to `router.post('/:id/undo', ...)`. GT16 asserts 401 without cookie. |
| T-02-04 | Tampering | Emit before DB write (race / inconsistency) | mitigate | All emit calls placed AFTER the DB INSERT/DELETE and AFTER `activeGames.set`. Order in code: INSERT/DELETE → applyThrow OR reconstructState → activeGames.set → if(io) emit. ST02 asserts the emit fires AFTER the throw is persisted (the await fetch resolves before the event arrives). |
| T-02-04b | Tampering | game_players.role injection from request body | mitigate | role is treated as opaque text and only consumed by `fuchsjagd.initState` which compares against the literal `'fuchs'`. No SQL/JS interpolation outside prepared statements. Invalid roles are simply ignored by game-type modules. |
| T-02-05 | Information Disclosure | reconstructState reads meta column | accept | meta values are produced server-side from request bodies that already pass the route handler validation (Number.isInteger on value, etc.). No PII; opaque JSON. |
| T-02-06 | Denial of Service | Repeated undo on a game with one throw | accept | The 400 short-circuit guarantees no DB mutation when `!lastThrow`. No resource exhaustion vector. |
| T-02-Inj | Injection | DELETE FROM throws WHERE id = ? | mitigate | Prepared statement; `lastThrow.id` is read from a parameterized SELECT. No string concatenation. |
</threat_model>
