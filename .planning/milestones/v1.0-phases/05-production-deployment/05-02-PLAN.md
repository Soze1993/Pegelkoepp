<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (HTTPS) → Nginx :443 | TLS terminates here; all public traffic enters through Nginx |
| Nginx → Node.js :3000 (localhost) | Nginx proxies over plain HTTP on loopback; X-Forwarded-* headers carry the original client context |
| SSH → VPS | Human operator access; guarded by SSH key; fail2ban optional |
| SQLite → backup directory | Read-only snapshot via VACUUM INTO; backup dir is outside the git repo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Information Disclosure | HTTP traffic (no TLS) | mitigate | Nginx port-80 block uses `return 301 https://$host$request_uri` — all HTTP redirected to HTTPS |
| T-05-05 | Spoofing | WebSocket session (wss://) | mitigate | Socket.io cors: false (same-origin, confirmed in server.js line 26); Nginx Host header preservation (`proxy_set_header Host $http_host`) |
| T-05-06 | Elevation of Privilege | SSH brute-force | mitigate | ufw blocks all ports except 22/80/443; fail2ban optional additional layer |
| T-05-07 | Denial of Service | PM2 log disk fill | mitigate | pm2-logrotate: max_size 10M, retain 7 days — configured in DEPLOY.md Section 7b |
| T-05-08 | Tampering | Static files served via Nginx alias | accept | Nginx `alias` directive scoped to `public/` directory only; no `root /` traversal risk |
| T-05-09 | Information Disclosure | .env on VPS | mitigate | .env is in .gitignore (already verified in Plan 01); DEPLOY.md warns never to commit it; file permissions should be 600 (chmod 600 .env — note in DEPLOY.md Section 5) |
| T-05-SC | Tampering | system apt packages (nginx, certbot, ufw, sqlite3) | accept | All are canonical Ubuntu packages from official apt repos; no npm packages installed in this plan |
</threat_model>
