<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Nginx (HTTPS) | Untrusted client input arrives over TLS; Nginx terminates TLS |
| Nginx → Node.js (HTTP/1.1 localhost) | Internal proxy hop; app must trust X-Forwarded-* headers from Nginx |
| Node.js → SQLite | Localhost only; app is the only writer |
| DEPLOY.md on disk | Contains shell commands that run as app user — no secrets, but must not contain PIN values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | session cookie over HTTP | mitigate | `app.set('trust proxy', 1)` enables `cookie.secure:true` behind Nginx; without it, cookies are sent over HTTP |
| T-05-02 | Tampering | ecosystem.config.js in git | accept | File contains no secrets (PIN_HASH and SESSION_SECRET are loaded from .env via dotenv, not from this file); low-value tampering surface |
| T-05-03 | Information Disclosure | DEPLOY.md in git | accept | Contains commands and patterns, no actual secrets; instructs human to never commit .env |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new npm packages installed in this plan (ecosystem.config.js is a config file, not a dependency); all new packages are system apt packages addressed in Plan 02 |
</threat_model>
