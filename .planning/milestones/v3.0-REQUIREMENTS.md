## v3.0 Requirements

### PROFILE — Profilbilder

- [x] **PROFILE-01:** Benutzer kann für jeden Spieler ein Profilbild hochladen (JPEG/PNG, max 5 MB)
  - *Outcome: POST /api/players/:id/photo mit express.raw, magic-byte-Validierung, 5MB limit — Phase 11 Plan 01*
- [x] **PROFILE-02:** Profilbild wird in der Spielerliste und im Spielerprofil (Tablet) angezeigt
  - *Outcome: img+emoji Overlay in renderSpielerListe und showSpDetail, uploadPhoto() global — Phase 11 Plan 02*
- [x] **PROFILE-03:** Profilbild erscheint auf dem TV-Screen (Idle-Screen + neben aktivem Spieler im laufenden Spiel)
  - *Outcome: tvPlayers cache, renderPlayerGrid() 80px Idle-Grid, 40px Avatar in Generic + alle 7 Spieltyp-Renderer — Phase 11 Plan 03*
- [x] **PROFILE-04:** Kein Profilbild → Emoji-Fallback (kein leeres Feld, kein broken image icon)
  - *Outcome: onerror="this.style.display='none'" Muster — img versteckt sich, emoji wird sichtbar — Phases 11 Plan 02+03*

### GUEST — Gastkegler

- [x] **GUEST-01:** Benutzer kann einen Gastkegler (Name + Emoji) für eine Spielsitzung erstellen
  - *Outcome: Gast-Checkbox in Spieler-Dialog, async addSpieler() mit is_guest:true — Phase 10 Plan 02*
- [x] **GUEST-02:** Gastkegler ist in der Spielerauswahl mit "(Gast)"-Kennzeichnung sichtbar und spielbar wie ein regulärer Spieler
  - *Outcome: (Gast) ternary in 5 render-Funktionen (Spielerliste, BK, VG, FJ×2) — Phase 10 Plan 02*
- [x] **GUEST-03:** Gastkegler werden aus allen Statistiken und dem Leaderboard ausgeschlossen
  - *Outcome: AND is_guest = 0 in allen Player-SELECT-Queries in stats.js — Phase 10 Plan 01*
- [x] **GUEST-04:** Gastkegler werden beim Beenden eines Kegelabends automatisch archiviert
  - *Outcome: UPDATE players SET archived=1 WHERE is_guest=1 in POST /api/abende/:id/end — Phase 10 Plan 02*

### STATS — Team-Gewinner

- [x] **STATS-01:** Viergewinnt- und Fuchsjagd-Siege werden als individuelle Siege für alle Teammitglieder gezählt
  - *Outcome: winners.length === 0 als isDraw, for..of winners loop für multi-winner, winners.some() für H2H — Phase 10 Plan 01*
- [x] **STATS-02:** Historische VG/FJ-Spiele werden rückwirkend korrekt gewertet (Stats recomputed)
  - *Outcome: Stats sind query-time computed — kein Backfill nötig, historische Spiele korrekt nach Neustart — Phase 10 Plan 01*

### TVFIX — TV Layout Fixes

- [x] **TVFIX-01:** Fuchsjagd TV skaliert korrekt für bis zu 11 Jäger
  - *Outcome: position:fixed + flex:1 rows, 3-stufiges Scaling (≤6/7–9/≥10), Fuchs-Avatar immer 80px — Phase 12*
- [x] **TVFIX-02:** KDA TV: Der Großes-Finale-Slot wird erst eingeblendet wenn beide Finalisten feststehen
  - *Outcome: gfVisible = !!(gfSlot && gfSlot.p1 && gfSlot.p2) — kein leerer GF-Slot mehr — Phase 12*
