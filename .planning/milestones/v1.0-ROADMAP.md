# Milestone v1.0: MVP — App Live

**Status:** ✅ SHIPPED 2026-05-22  
**Phases:** 1–5  
**Total Plans:** 15  
**Production URL:** https://kegel.pegelkoepp.de

## Overview

v1.0 delivered a full-stack club bowling app for Pegelköpp — a German Kegelclub. The existing `kegelclub_12.html` single-page frontend was wired to a persistent Node.js/Express backend with SQLite, real-time Socket.io sync, and deployed to a Netcup VPS with HTTPS, PM2, and Nginx.

The core value was achieved: a throw entered on a tablet appears on the TV screen in under 2 seconds via WebSocket — live, without page reload.

## Milestone Summary

**Git Range:** 72 commits (2026-05-20 → 2026-05-22)  
**Files changed:** 50 files, ~11,000 lines added

**Duration:** 3 days

**Key Decisions:**

| Decision | Rationale |
|----------|-----------|
| Node.js + Express + SQLite (better-sqlite3) | Match existing JS codebase; SQLite is right-sized for single-club single-writer |
| Socket.io 4 over native ws | Auto-reconnect, rooms, HTTP fallback — required for TV reliability |
| DB-first throw persistence | Crash recovery is non-negotiable: INSERT before applyThrow |
| Existing kegelclub_12.html evolved (not rewritten) | Design + game logic already complete; minimize risk |
| textContent-only XSS guard in TV and frontend | Hard constraint; no innerHTML for any DB-sourced string |
| SQLite WAL mode from day one | Two-line fix; skip at your peril (concurrent reads) |
| Trust proxy = 1 for Nginx | Enables Secure cookie flag when behind reverse proxy |
| PM2 fork mode (not cluster) | SQLite is single-writer; cluster adds WAL contention |
| Running as root on VPS (pm2-root.service) | Netcup Debian 12 default; log paths use /root/.pm2/logs/ |

**Issues Resolved:**

- Helmet CSP blocked inline script on index.html → added `unsafe-inline` to script-src
- TV not receiving throw events (TV socket never joined game room) → added server-side `join` handler
- GET /api/games missing abend_id in SELECT → added to both queries in games.js
- Google Fonts not rendering on TV (offline-risk avoidance) → added CDN link with hard pixel fallback

**Technical Debt Incurred:**

- Self-hosting Bebas Neue / DM Sans (currently CDN — offline venue risk)
- TV font readability at true 5m venue distance needs confirmation at first Kegelabend
- Geteilte game type score shapes on TV are generic wuerfe.reduce (good enough for now)
- Monthly leaderboard, win streaks, WhatsApp share deferred to v2.0
